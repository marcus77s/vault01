## 🏛️ As APIs oficiais que existem hoje

### 1\. **PNCP – Portal Nacional de Contratações Públicas** ⭐ (a mais importante pra você)

É o portal oficial criado pela Lei 14.133. [16](https://gist.github.com/Micael106/04a3e5515057ab11ea8797603682f0bd)O PNCP é o sítio oficial estabelecido pela Lei 14.133 para divulgação e gestão de contratações públicas no Brasil, centralizando informações, editais e contratos, e é gerido por um comitê nacional.

- **Swagger de consulta:** `pncp.gov.br/api/consulta/swagger-ui/index.html`
- [16](https://gist.github.com/Micael106/04a3e5515057ab11ea8797603682f0bd) As consultas cobrem contratações, alienação de bens móveis e imóveis, atas de registro de preços e contratos no âmbito da Lei 14.133/2021, via API REST com dados em JSON.

Ponto crucial para o seu projeto: [17](https://repositorio.ufsc.br/bitstream/item/51718abf-525e-4867-a7dd-035657231915/Manual%20de%20Integra%C3%A7%C3%A3o%20PNCP%20%E2%80%93%20Vers%C3%A3o%201.0.0.pdf?sequence=1)o acesso ao portal de consultas é público, mas as APIs de manutenção (inserção, retificação ou exclusão de dados) requerem autenticação/autorização. Ou seja, **ler editais/contratos = livre; publicar = precisa credenciamento.**

E sem burocracia para ler: [13](https://www.gov.br/pncp/pt-br/acesso-a-informacao/dados-abertos)todas as buscas podem ser feitas sem precisar se cadastrar ou fazer login, num esforço para tornar os dados das compras públicas mais acessíveis de forma rápida e democrática.

**Exemplo real de endpoint de consulta** (visto em uso na comunidade):

```
GET https://pncp.gov.br/api/consulta/v1/contratacoes/publicacao
    ?dataInicial=20240625
    &dataFinal=20240626
    &codigoModalidadeContratacao=6   # 6 = pregão eletrônico
    &uf=AL
    &tamanhoPagina=50
    &pagina=1
```

Retorna JSON com órgão, CNPJ, modalidade, sequencial, datas etc. — perfeito para alimentar seu **RAG 1 (editais OK)** e **RAG 2 (padrões de falha)**.

### 2\. **dadosabertos.compras.gov.br** ⭐ (CATMAT / CATSER — ouro pra você)

Essa é a API que vai resolver sua **biblioteca de Especificações Técnicas por catálogo** que falei antes.

- **Swagger:** `dadosabertos.compras.gov.br/swagger-ui/index.html`
- [10](https://documenter.getpostman.com/view/13166820/2sA3XJjPpR) O CATMAT (Catálogo de Materiais) e o CATSER (Catálogo de Serviços) do SIASG são as bases que identificam todos os materiais e serviços licitados pela Administração Pública Federal, e todas as operações no Compras.gov.br usam esses catálogos para definir os objetos das licitações.

Repara que **já tem grupo militar catalogado** — exemplo real do retorno: [10](https://documenter.getpostman.com/view/13166820/2sA3XJjPpR)consultando o código de grupo 16, a API retorna "COMPONENTES E ACESSORIOS DE AERONAVES". Isso conecta direto com a parte de defesa do seu projeto.

Exemplo:

```
curl 'https://dadosabertos.compras.gov.br/modulo-material/1_consultarGrupoMaterial?codigoGrupo=16'
```

### 3\. **compras.dados.gov.br** (API legada do SIASG)

A API antiga, ainda no ar, baseada em REST/HATEOAS. [4](https://compras.dados.gov.br/docs/home.html)São os Dados Abertos do SIASG, o sistema onde se operacionalizam as compras do Governo Federal.

- [4](https://compras.dados.gov.br/docs/home.html) Você pode navegar por todos os recursos (fornecedores, serviços, materiais, licitações, contratos) apenas usando os links disponíveis.
- [4](https://compras.dados.gov.br/docs/home.html) Os dados saem em JSON, XML e CSV (basta trocar "html" pelo formato) e são disponibilizados sob a licença ODBL (Open DataBase License).

⚠️ Atenção: é a geração **antiga**. Para projeto novo, priorize PNCP + dadosabertos.compras.gov.br. Use essa legada só para dados históricos que não estejam nas novas.

## ⚠️ O que essas APIs **NÃO** te dão (importante!)

Você mencionou "modelos, leis etc". Cuidado com a expectativa:

| Você quer | A API entrega? |
| --- | --- |
| **Dados** de editais/contratos/preços | ✅ Sim, PNCP + dadosabertos |
| **Catálogo** de materiais/serviços (CATMAT/CATSER) | ✅ Sim, com grupos militares inclusive |
| **Texto integral das leis** (14.133, 8.666) | ❌ Não — isso vem do Planalto/LexML |
| **Minutas-modelo da AGU** | ❌ Não — vem do site da AGU separadamente |
| **Acórdãos do TCU** | ❌ Não — API/busca do próprio TCU |
| **Normas militares (12.598, IG 10-01)** | ❌ Não — coleta manual nas Forças |

Então o PNCP/Compras alimenta seus **RAG 2 e RAG 3 (dados reais e exemplos)**, mas o **RAG 1 (base legal)** e o **RAG militar** você monta de outras fontes:

- **Leis:** LexML (`lexml.gov.br`) tem API e é a fonte canônica versionada
- **Jurisprudência TCU:** Pesquisa Integrada do TCU tem endpoints próprios
- **Minutas AGU:** portal da AGU (modelos de editais e contratos)

## 💡 Dicas práticas de quem vai integrar

1. **Paginação obrigatória** — o PNCP devolve em páginas (`tamanhoPagina`, `pagina`). Os exemplos usam blocos de 50 registros. Monte loop com controle de `paginasRestantes`.
2. **Filtre por CNPJ dos órgãos militares** — para puxar só editais do EB/MB/COMAER, filtre pelo CNPJ/UASG das organizações militares. Assim você popula direto o "RAG de contexto institucional" que sugeri antes.
3. **Código de modalidade importa** — no PNCP, `codigoModalidadeContratacao=6` é pregão eletrônico. Mapeie as modalidades relevantes pra defesa (incluindo dispensa/inexigibilidade).
4. **Guarde o `dataAtualizacao`** — o retorno traz timestamp de atualização. Use isso para ingestão incremental (só puxa o que mudou).
5. **Licença ODBL** — os dados do compras.dados.gov.br exigem atribuição/compartilhamento na mesma licença. Documenta isso no seu projeto, especialmente se virar produto.

## 🎯 TL;DR

- **PNCP** (`pncp.gov.br/api/consulta`) = sua fonte principal de editais/contratos reais, **leitura pública sem login**.
- **dadosabertos.compras.gov.br** = CATMAT/CATSER, ótimo para suas **ETs militares** (já tem grupos de aeronáutica, armamento etc.).
- **compras.dados.gov.br** = legada, use só para histórico.
- **Leis e minutas-modelo NÃO vêm dessas APIs** — você puxa de LexML, TCU e AGU separadamente.

### 🔌 Como plugar as APIs do Governo nos seus 3 RAGs

#### 1\. RAG 1: Base Legal (Leis e Normas)

- **A Fonte:** API do LexML (Senado/Câmara) e Sistema de Normas.
- **Como usar:** Em vez de fazer upload manual de PDFs das leis, seu sistema consome a API para garantir que se a Lei 14.133 for alterada hoje à tarde, seu sistema sabe amanhã de manhã.
- **O pulo do gato para FA:** Você pode criar um *crawler* que baixa as portarias específicas do Ministério da Defesa, EB, MB e FAB publicadas no Diário Oficial (que também tem API no Imprensa Nacional).

#### 2\. RAG 2: Padrões de Falha (Histórico e Riscos)

- **A Fonte:** API do PNCP (Swagger PNCP) + API de Jurisprudência do TCU.
- **Como usar:** A API do PNCP retorna o *status* de cada licitação. Seu sistema pode puxar automaticamente todo edital que teve o status **"Anulada"**, **"Revogada"** ou **"Suspensa"**.

6. Pesquisa de preços
7. **Checklist de conformidade** assinado pelo sistema (mostrando que passou em N validações)

Esse último item é o que vende. O ordenador de despesas assina mais tranquilo sabendo que tem um relatório de auditoria prévia.

A situação do Linux em meio a essa novidade do **Nvidia RTX Spark** é fascinante, mas apresenta um cenário de amor e ódio para os usuários do pinguim. A história se divide em duas realidades muito distintas: o mercado corporativo/desenvolvedores e o mercado consumidor.

Para entender onde o Linux fica, precisamos olhar para os detalhes desse lançamento e para a aliança pesada entre a Nvidia e a Microsoft. Aqui estão os pontos fundamentais sobre o futuro do Linux com esses chips:

**1\. A Versão "Pro" já roda Linux nativamente (DGX Spark)** A arquitetura do RTX Spark na verdade nasceu de um projeto voltado para cientistas de dados, chamado **Nvidia DGX Spark**. Esse mini-PC (que chega a custar milhares de dólares) compartilha praticamente o mesmo silício, mas **roda uma versão customizada do Ubuntu Linux**. Ou seja, no nível de hardware, os drivers existem e o chip da Nvidia em parceria com a MediaTek é plenamente capaz de rodar Linux maravilhosamente bem.

**2\. Os Notebooks de Consumo nascem focados no Windows (A Ameaça do "Lock-in")** O problema é que os notebooks (como o Microsoft Surface, Dell XPS, HP OmniBook, etc.) que receberão o **RTX Spark** no final de 2026 foram desenhados em uma parceria profunda com a Microsoft para serem a plataforma definitiva dos **Copilot+ PCs**. Para os consumidores finais, o sistema estará extremamente "amarrado" ao Windows 11. Isso significa que, assim como ocorreu com os chips Snapdragon X Elite e com o Apple M1/M2/M3, instalar um Linux nativo nestes notebooks logo no lançamento será uma dor de cabeça enorme. Inicialmente, faltarão drivers de código aberto para gerenciar a energia do chip, a memória unificada e componentes específicos das placas-mãe.

**3\. O papel salvador do WSL2 (Windows Subsystem for Linux)** A Nvidia sabe que quem trabalha de verdade com Inteligência Artificial, desenvolvimento em CUDA (que roda nativamente no RTX Spark) e servidores **usa Linux**. A saída estratégica de Nvidia e Microsoft foi garantir que o **WSL2** rodasse de forma impecável nesses chips. Assim, um desenvolvedor pode ter o ambiente Windows gerenciando o hardware proprietário, enquanto roda seu Ubuntu via WSL acessando diretamente todos os 6.144 núcleos CUDA e a RAM de 128 GB sem gargalos aparentes.

**4\. A comunidade Open-Source e a "Saga Asahi 2.0"** Com o tempo, os puristas do Linux certamente vão querer formatar o Windows e rodar um Fedora, Arch ou Ubuntu nativo. Muito provavelmente veremos o surgimento de um grande esforço da comunidade de engenharia reversa (parecido com o que o projeto *Asahi Linux* fez para levar o pinguim aos Macs com Apple Silicon). A dificuldade aqui será fazer a integração open-source funcionar bem com o componente gráfico da Nvidia (Blackwell) associado ao processador ARM da MediaTek.

**5\. E para quem joga no Linux? (A questão do Proton/SteamOS)** Um grande atrativo do Linux hoje em dia são os jogos, impulsionados pela Valve e o Steam Deck. Porém, o RTX Spark usa arquitetura **ARM**, e 99% dos jogos de PC são **x86** (Intel/AMD). No Windows, a Microsoft usará seu próprio emulador (Prism) para traduzir o código do jogo na hora. No Linux, a comunidade ainda precisaria amadurecer bastante ferramentas de tradução x86-para-ARM (como o *FEX-Emu* ou *Box64*) para rodar jogos pesados nesses chips ARM da Nvidia com desempenho aceitável.

**Em resumo:** O hardware do RTX Spark ama o Linux (e os desenvolvedores de IA exigem isso), mas os notebooks de prateleira foram programados para **fortalecer o Windows**. Se você for um desenvolvedor, provavelmente usará Linux através de virtualização (WSL2) no começo. Se quiser "formatar e instalar" um Linux de forma nativa e pura em um laptop RTX Spark, terá que esperar a comunidade open-source criar soluções, o que pode levar meses ou até anos após o lançamento.

Sim, a afirmação está corretíssima. Recentemente (no final de maio e início de junho de 2026, durante a Computex), a Nvidia abalou o mercado de tecnologia ao anunciar o **Nvidia RTX Spark**, um novo superchip ("System on a Chip" - SoC) voltado para notebooks Windows e PCs compactos que entra em rota de colisão direta com os processadores Apple Silicon (como o M5 Max) que equipam os MacBooks.

Essa é a primeira grande investida da Nvidia no mercado de processadores completos para computadores de consumo e representa a maior "ameaça" ao domínio de performance e eficiência que a Apple construiu nos últimos anos.

Aqui está o porquê de o RTX Spark ser visto como um "assassino de MacBook Pro" e uma ameaça à Apple:

**1\. A Mesma Estratégia da Apple (Só que com o poder de GPU da Nvidia)** Assim como os chips da série "M" da Apple, o RTX Spark é baseado na arquitetura **ARM**.[1](https://www.macworld.com/article/3156077/nvidias-rtx-spark-laptops-are-gunning-for-the-macbook-pro-yawn.html)[2](https://www.macrumors.com/2026/06/01/nvidia-challenges-apple-rtx-spark-pc-chip/) Desenvolvido em forte parceria com a MediaTek, o chip unifica um processador de altíssima eficiência (CPU de 20 núcleos) com uma poderosa placa de vídeo **Nvidia Blackwell RTX** de até 6.144 núcleos CUDA.[3](https://www.mediatek.com/products/personal-computing/nvidia-rtx-spark)

**2\. Memória Unificada de até 128 GB** A Nvidia adotou a grande sacada dos Macs: a memória unificada (onde CPU e GPU dividem a mesma memória de alta velocidade). O RTX Spark suporta até 128 GB de RAM unificada através da tecnologia NVLink.[1](https://www.macworld.com/article/3156077/nvidias-rtx-spark-laptops-are-gunning-for-the-macbook-pro-yawn.html)[4](https://indianexpress.com/article/explained/explained-sci-tech/nvidia-rtx-spark-ai-pc-chip-apple-intel-10719709/) Isso permite que notebooks finos consigam rodar modelos de Inteligência Artificial gigantes (de até 120 bilhões de parâmetros) e renderizações de altíssima resolução totalmente locais, sem depender da nuvem.

**3\. Desempenho Monstruoso em IA Local** O RTX Spark foi projetado como o computador para a "Era dos Agentes Pessoais de IA".[3](https://www.mediatek.com/products/personal-computing/nvidia-rtx-spark)[5](https://nvidianews.nvidia.com/news/nvidia-microsoft-windows-pcs-agents-rtx-spark) Ele entrega impressionante **1 Petaflop de desempenho de IA**.[3](https://www.mediatek.com/products/personal-computing/nvidia-rtx-spark)[6](https://www.nvidia.com/en-us/products/rtx-spark/) Com isso, ele desbanca outras soluções da Intel, AMD e até os próprios chips Qualcomm e Apple, permitindo que ferramentas como Photoshop, Premiere, ComfyUI e agentes do Copilot+ rodem nativamente e muito mais rápido.

**4\. Bateria que dura o dia todo** Historicamente, colocar uma placa de vídeo dedicada potente da Nvidia em um notebook Windows significava ter um aparelho pesado e com bateria que durava apenas poucas horas. Como o RTX Spark é um chip ARM altamente otimizado de baixo consumo (com o TDP operando ali na faixa dos 45W a 80W), ele promete eficiência energética e duração de bateria parecidas com as do MacBook Pro.

**Resumo da Ópera:** A Nvidia, junto com a Microsoft, não quer apenas competir no mundo dos games; elas querem os profissionais criativos e desenvolvedores de volta para o Windows. Ao empacotar a supremacia da Nvidia em Inteligência Artificial e processamento gráfico num chip ultrafino com arquitetura ARM, o RTX Spark força a Apple a enfrentar, pela primeira vez desde que abandonou a Intel, uma concorrência à altura no seu principal território: laptops potentes, frios, finos e com bateria para o dia inteiro. Laptops com esse chip de empresas como Asus, Dell, Lenovo e a própria linha Microsoft Surface começarão a chegar ao mercado ainda este ano.[2](https://www.macrumors.com/2026/06/01/nvidia-challenges-apple-rtx-spark-pc-chip/)[5](https://nvidianews.nvidia.com/news/nvidia-microsoft-windows-pcs-agents-rtx-spark)

### 🔌 Como plugar as APIs do Governo nos seus 3 RAGs

#### 1\. RAG 1: Base Legal (Leis e Normas)

- **A Fonte:** API do LexML (Senado/Câmara) e Sistema de Normas.
- **Como usar:** Em vez de fazer upload manual de PDFs das leis, seu sistema consome a API para garantir que se a Lei 14.133 for alterada hoje à tarde, seu sistema sabe amanhã de manhã.

#### 2\. RAG 2: Padrões de Falha (Histórico e Riscos)

- **A Fonte:** API do PNCP (Swagger PNCP) + API de Jurisprudência do TCU.
- **Como usar:** A API do PNCP retorna o *status* de cada licitação. Seu sistema pode puxar automaticamente todo edital que teve o status **"Anulada"**, **"Revogada"** ou **"Suspensa"**.

#### 3\. RAG 3: Templates Fix (Modelos OK)

- **A Fonte:** Modelos da AGU (Advocacia-Geral da União).
- **O detalhe técnico:** A AGU publica os "Modelos de Licitação" não como texto puro em API, mas como arquivos DOCX/PDF no site deles.

### 🪖 O "Killer Feature" para as Forças Armadas usando a API

No sistema Compras.gov.br, cada Órgão/Quartel tem um código chamado **UASG** (Unidade Administrativa de Serviços Gerais). O Ministério da Defesa e as Forças Armadas têm faixas específicas de UASG (ex: UASGs do Exército geralmente começam com 16, Marinha com 7, Aeronáutica com 12 - *valores ilustrativos, variam por OM*).

Você pode configurar seu sistema para bater na API do Governo assim:

```
GET https://pncp.gov.br/api/pncp/v1/orgaos/{CNPJ_DA_OM}/compras
```

**O que isso permite fazer no seu sistema:**

1. **Clonagem do Bem (Benchmarking Militar):** O encarregado diz: *"Preciso comprar ração operacional"*. O sistema bate na API, filtra pelas UASGs do Exército, pega os 10 últimos editais de ração operacional que deram **SUCESSO** e gera um template perfeito baseado neles.
